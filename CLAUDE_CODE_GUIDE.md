# Claude Code を使ったローカル編集ワークフロー

このガイドでは、構築された「Nanami's Secret Base」ファンクラブサイトを、ローカル環境でAnthropicの **Claude Code** を使用して編集・拡張していくためのワークフローを説明します。

## 1. 準備：リポジトリのクローン

まずは、作成されたGitHubリポジトリをローカル環境にクローンします。

```bash
# ターミナルで実行
git clone https://github.com/nobu0917/nanami-secret-base.git
cd nanami-secret-base
```

## 2. Claude Code の起動

プロジェクトディレクトリ内でClaude Codeを起動します。

```bash
# Claude Codeを起動
claude
```

## 3. デザインシステムの共有（重要）

Claude Codeに一貫したデザインで編集させるために、まずはデザインのルール（デザインシステム）を伝えます。
以下のプロンプトをClaude Codeに入力してください。

> **プロンプト例：**
> 「このプロジェクトのCSS（`assets/css/style.css`）を確認してください。このサイトは『スノーホワイト（ベース）＋パステルピンク（アクセント）＋ダークブルー（テキスト）』を基調とした、清潔感と暖かさのあるデザインです。
> 今後の編集や新しいページの作成では、このCSSで定義されているCSS変数（`--color-pink`、`--color-blue-dark`など）や、既存のコンポーネントクラス（`.btn`, `.section-title`など）を再利用して、デザインの一貫性を保ってください。」

## 4. 編集の依頼方法（ユースケース別）

Claude Codeに具体的な編集を依頼する際のプロンプトのコツです。

### ケースA：既存ページの文言や画像の差し替え
具体的なファイル名と変更箇所を指示します。

> **プロンプト例：**
> 「`index.html` の NEWS セクションにある最新ニュースの日付を『2026.04.01』に、タイトルを『ファンクラブ限定ライブ開催決定！』に変更してください。」

### ケースB：新しいセクションの追加
既存のクラスを活用するよう明記します。

> **プロンプト例：**
> 「`index.html` の STORE セクションの下に、新しく『DISCOGRAPHY（ディスコグラフィー）』セクションを追加してください。
> デザインは既存の `.section` や `.section-title` クラスを使用し、3枚のCDジャケット画像が並ぶグリッドレイアウト（`.store-preview-grid` のような構造）にしてください。」

### ケースC：新しいページの作成
既存のページをテンプレートとして使うよう指示します。

> **プロンプト例：**
> 「`news.html` をベースにして、新しく `profile.html`（ななみのプロフィールページ）を作成してください。
> ヘッダーとフッターは共通のままにし、メインコンテンツ部分に、ななみの全身画像（`assets/images/nanami_main.jpg`）と、自己紹介文、Q&Aリストを配置してください。」

## 5. レビューと調整

Claude Codeがコードを変更したら、ローカルでHTMLファイルを開いてブラウザで確認します。
デザインが崩れていたり、意図と違う場合は、具体的に修正を指示します。

> **修正プロンプト例：**
> 「追加してくれたボタンですが、ピンクのグラデーションになっていません。`assets/css/style.css` にある `.btn-primary` クラスを適用してください。」

## 6. 変更の保存（Git）

編集が完了し、ブラウザでの確認もOKなら、Gitで変更をコミットしてGitHubにプッシュします。
これもClaude Codeに依頼できます。

> **プロンプト例：**
> 「今の変更内容をコミットして、GitHubにプッシュしてください。コミットメッセージは『プロフィールページの追加とナビゲーションの更新』としてください。」

---

## 💡 Claude Codeを使いこなすコツ

1. **小さく頼む**: 一度に「サイト全体をリニューアルして」と頼むより、「まずヘッダーのメニューを追加して」「次にフッターの色を変えて」と段階的に頼む方が正確です。
2. **既存コードを参照させる**: 「〇〇ページと同じレイアウトで」と指示すると、サイト全体の一貫性が保たれます。
3. **迷ったら質問させる**: プロンプトの最後に「実装を始める前に、不明点があれば質問してください」と付け加えると、認識のズレを防げます。
